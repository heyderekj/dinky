# Welcome

Dinky 使文件變得很小。放入一些東西，取出一個較小的版本——外觀相同，重量更輕。

它適用於**圖像**（JPEG、PNG、WebP、AVIF、TIFF、BMP）、**視訊**（MP4、MOV、M4V）和**PDF**。

一切都發生在您的 Mac 上。沒有上傳任何內容。

---

## Quick start

1. 將一個檔案（或一堆檔案）拖曳到 Dinky 的視窗上。
2. 觀察計數下降。
3. 在您的下載資料夾或您選擇的資料夾中找到原件旁邊（預設）的較小副本。

就是這樣。預設值很好。如果您想讓它們屈服於您的意願，請繼續閱讀。

## Ways to compress

您不必先打開應用程式。選擇適合您工作方式的選項。

- **拖放**到 Dinky 視窗或 Dock 圖示上。
- **開啟檔案…** — `{{SK_OPEN_FILES}}` 從工作表中選擇。
- **剪貼簿壓縮** — `{{SK_PASTE}}` 貼上在 Finder 中複製的受支援的**檔案**（圖像、影片、PDF）或**原始圖像**資料（來自螢幕截圖或瀏覽器的 PNG/TIFF）。
- **右鍵點選 Finder → 服務 → 使用 Dinky 壓縮** — 適用於任何大小的選擇。
- **觀看資料夾** — Dinky 會壓縮其中的任何新內容。 （請參閱下面的*監視資料夾*。）
- **快速操作** — 在系統設定 → 鍵盤 → 鍵盤快速鍵 → 服務中為「使用 Dinky 壓縮」指派鍵盤快速鍵。

## Where files go

在 **設定 → 輸出** 中設定一次。

- **與原始資料夾相同** *（預設）* — 保持內容整潔和本地化。
- **下載資料夾** — 如果您處理大量電子郵件或訊息，則非常有用。
- **自訂資料夾...** — 將 Dinky 指向任何地方。

### Filenames

- **附加「-dinky」** *（預設）* — `photo.jpg` 變成 `photo-dinky.jpg`。原件是安全的。
- **取代原始** — 覆蓋檔案。如果您希望最後得到一個乾淨的文件，請與 **常規** 中的 *將原始文件移至垃圾箱* 結合使用。
- **自訂後綴** — 用於工作流程修補者。使用適合您的檔案系統的檔案系統。

> **專業提示：** 預設可以覆寫每個規則的儲存位置和檔案名稱。將其用於「螢幕截圖 → `~/Desktop/web/`，取代原始內容」之類的內容。

## Sidebar & formats

主視窗右側的側邊欄是您告訴 Dinky **要做什麼**的地方。

### Simple sidebar (default)

三種簡單語言選擇：**圖像**、**影片**、**PDF**。每個類別選擇一個並刪除。 Dinky 找出了一個合理的編碼器、品質和尺寸。

### Full sidebar

關閉 **設定 → 常規 → 使用簡單側邊欄**（或開啟各個部分）以顯示每個控制項：

- **圖像** — 格式、內容提示（照片/插圖/螢幕截圖）、最大寬度、最大檔案大小。
- **影片** — 編解碼器系列 (H.264 / HEVC / AV1)、品質層、條帶音訊。
- **PDF** — 保留文字和鏈接，或拼合為圖像以獲得盡可能小的文件。

## Smart quality

當**智慧品質**開啟時（新預設的預設設定），Dinky 會檢查每個檔案並為其選擇設定：

- 影像的編碼器會根據其內容進行調整（繁忙的照片與圖形 - UI、插圖、標誌、螢幕截圖）。
- 影片根據解析度和來源位元率獲得一個等級，然後根據內容類型進行調整 - 螢幕錄製和動畫/動態圖形會上升一個等級，以便文字和邊緣保持可讀。攝影機鏡頭透過 EXIF 品牌/型號進行識別，因此不會受到過度保護。 HDR 來源（杜比視界、HDR10、HLG）使用 HEVC 匯出，以保留顏色並突出顯示細節； H.264 會默默地將它們扁平化為 SDR。
- PDF 根據文件複雜性以及它們是文字優先還是圖像為主而獲得一個等級。

當您需要固定品質層（影片為平衡/高，PDF 為低/中/高）時，請在**壓縮**下的任何預設中將其關閉 - 對於需要可預測結果的批次非常有用。

## Presets

預設是已儲存的設定組合。為每個重複任務建立一個。

效果良好的範例：

- **Web 英雄圖像** — WebP，最大寬度 1920，附加 `-web`。
- **客戶端可交付成果** — WebP，最大寬度 2560，取代原始內容，儲存至「~/Deliverables/」。
- **螢幕錄製** — H.264 平衡、帶狀音訊。
- **掃描的 PDF** — 平整、中等品質、灰階。

在**設定 → 預設**中建立它們。每個都可以：

- 適用於所有媒體或僅一種類型（圖像/視訊/PDF）。
- 使用自己的儲存位置和檔案名稱規則。
- 觀看自己的資料夾*（見下文）*。
- 刪除元數據，清理檔案名，完成後開啟輸出資料夾。

Switch the active preset from the sidebar at any time.

## Watch folders

將檔案放入資料夾中，讓 Dinky 在背景處理它們。

- **全球觀看** — *設定→觀看→全球*。使用側邊欄目前設定的任何內容。適用於「傳入」或螢幕截圖資料夾。
- **按預設觀看** — 每個預設也可以使用自己的規則觀看自己的資料夾。獨立於側邊欄－隨心所欲地改變側邊欄，預設仍可發揮作用。

> **專業提示：** 結合「螢幕錄製資料夾」+ 一個可剝離音訊並重新編碼為 H.264 平衡的預設。點擊“⌘⇧5”，螢幕錄製，點擊停止——在您到達 Finder 之前，Dinky 已準備好一個小檔案。

## Manual mode

當您想要完全控制時，開啟**設定 → 常規 → 手動模式**。

放入的檔案不會自動壓縮。右鍵點選任意行即可當場選擇格式，當佇列準備就緒時使用 **檔案 → 立即壓縮** (`{{SK_COMPRESS_NOW}}`)，或先變更側邊欄中的設定。當一批包含非常不同的文件時很有用。

## Shortcuts app

Dinky 為「快捷方式」應用程式註冊了「壓縮圖片」操作。使用它透過 Dinky 以選定的格式傳輸 Finder 檔案或其他操作 - 與應用內壓縮相同的引擎（尊重智慧品質、調整大小和元資料的設定）。

## Privacy & safety

- 一切都在**本地運行**。沒有上傳，沒有遙測，沒有帳號。
- **崩潰報告**僅在*您*選擇發送時發送 - 通過崩潰後提示、“報告錯誤...”菜單或錯誤詳細信息表。如果您在系統設定中選擇了 macOS 診斷共享，Apple 也會透過 MetricKit 代表您向 Dinky 提供匿名崩潰數據，而不會留下任何其他數據離開您的 Mac。
- 編碼器（`cwebp`、`avifenc`、`oxipng` 以及 Apple 的內建 PDF 和 AVFoundation 視訊管道）內建於應用程式中並直接讀取您的檔案。
- 預設保留原件。 *壓縮後將原件移至垃圾箱*是可選的，位於**設定 → 常規**。
- *如果節省量低於則跳過*（預設關閉）可防止已經精簡的檔案被無用地重新編碼。
- 任何預設中的*剝離元資料*都會刪除 EXIF、GPS、相機資訊和顏色設定檔。在將照片發佈到網路之前值得這樣做。

## Troubleshooting

**產生的文件比原始文件大。 **
Dinky 則保留原件。您會在該行中看到*“無法將其縮小。保留原始版本。”*。

**文件被跳過。 **
要么它已經非常小（低於您的*如果節省低於*則跳過閾值），或者編碼器無法讀取它。按一下該行以了解詳細資訊。

**影片需要很長時間。 **
視訊重新編碼需要大量 CPU 資源。 **設定→常規**中的*批次速度*設定控制一次運行的檔案數量 - 如果您的 Mac 正在執行其他操作，請將其設定為 **快速**。

**我的 PDF 遺失了文字選擇/超連結。 **
您使用了*壓平（最小）*。將預設的 PDF 輸出切換為「保留文字和連結」並重新運行。扁平化總是以尺寸取勝；保留總是勝於實用性。

**右鍵點選「使用 Dinky 壓縮」未顯示。 **
安裝後打開 Dinky 一次，以便 macOS 註冊該服務。如果它仍然沒有出現，請在**系統設定→鍵盤→鍵盤快速鍵→服務→檔案和資料夾**中啟用它。

**為什麼Dinky不輸出JPEG？ **
WebP 和 AVIF 絕對優於 JPEG — 相同的視覺品質、更小的文件，並且在所有重要的地方都受到支援。如果您的平台需要 `.jpg`，請先嘗試 WebP；現在它幾乎已被普遍接受。如果您遇到真正拒絕它的地方，請聯繫我們並告知我們。
