# Welcome

Dinky 使文件变得很小。放入一些东西，取出一个较小的版本——外观相同，重量更轻。

它适用于**图像**（JPEG、PNG、WebP、AVIF、TIFF、BMP）、**视频**（MP4、MOV、M4V）和**PDF**。

一切都发生在您的 Mac 上。没有上传任何内容。

---

## Quick start

1. 将一个文件（或一堆文件）拖到 Dinky 的窗口上。
2. 观察计数下降。
3. 在您的下载文件夹或您选择的文件夹中找到原件旁边（默认）的较小副本。

就是这样。默认值很好。如果您想让它们屈服于您的意愿，请继续阅读。

## Ways to compress

您不必先打开应用程序。选择适合您工作方式的选项。

- **拖放**到 Dinky 窗口或 Dock 图标上。
- **打开文件…** — `{{SK_OPEN_FILES}}` 从工作表中选择。
- **剪贴板压缩** — `{{SK_PASTE}}` 粘贴在 Finder 中复制的受支持的**文件**（图像、视频、PDF）或**原始图像**数据（来自屏幕截图或浏览器的 PNG/TIFF）。
- **右键单击 Finder → 服务 → 使用 Dinky 压缩** — 适用于任何大小的选择。
- **观看文件夹** — Dinky 会压缩其中的任何新内容。 （请参阅下面的*监视文件夹*。）
- **快速操作** — 在系统设置 → 键盘 → 键盘快捷键 → 服务中为“使用 Dinky 压缩”分配键盘快捷键。

## Where files go

在 **设置 → 输出** 中设置一次。

- **与原始文件夹相同** *（默认）* — 保持内容整洁和本地化。
- **下载文件夹** — 如果您处理大量电子邮件或消息，则非常有用。
- **自定义文件夹...** — 将 Dinky 指向任何地方。

### Filenames

- **附加“-dinky”** *（默认）* — `photo.jpg` 变为 `photo-dinky.jpg`。原件是安全的。
- **替换原始** — 覆盖文件。如果您希望最后得到一个干净的文件，请与 **常规** 中的 *将原始文件移至垃圾箱* 结合使用。
- **自定义后缀** — 用于工作流程修补者。使用适合您的文件系统的文件系统。

> **专业提示：** 预设可以覆盖每个规则的保存位置和文件名。将其用于“屏幕截图 → `~/Desktop/web/`，替换原始内容”之类的内容。

## Sidebar & formats

主窗口右侧的侧边栏是您告诉 Dinky **做什么**的地方。

### Simple sidebar (default)

三种简单语言选择：**图像**、**视频**、**PDF**。每个类别选择一个并删除。 Dinky 找出了一个合理的编码器、质量和尺寸。

### Full sidebar

关闭 **设置 → 常规 → 使用简单侧边栏**（或打开各个部分）以显示每个控件：

- **图像** — 格式、内容提示（照片/插图/屏幕截图）、最大宽度、最大文件大小。
- **视频** — 编解码器系列 (H.264 / HEVC / AV1)、质量层、条带音频。
- **PDF** — 保留文本和链接，或拼合为图像以获得尽可能小的文件。

## Smart quality

当**智能质量**打开时（新预设的默认设置），Dinky 会检查每个文件并为其选择设置：

- 图像的编码器根据其内容进行调整（繁忙的照片与图形 - UI、插图、徽标、屏幕截图）。
- 视频根据分辨率和源比特率获得一个等级，然后根据内容类型进行调整 - 屏幕录制和动画/动态图形会上升一个等级，以便文本和边缘保持可读。摄像机镜头通过 EXIF 品牌/型号进行识别，因此不会受到过度保护。 HDR 源（杜比视界、HDR10、HLG）使用 HEVC 导出，以保留颜色并突出显示细节； H.264 会默默地将它们扁平化为 SDR。
- PDF 根据文档复杂性以及它们是文本优先还是图像为主而获得一个等级。

当您需要固定质量层（视频为平衡/高，PDF 为低/中/高）时，请在**压缩**下的任何预设中将其关闭 - 对于需要可预测结果的批次非常有用。

## Presets

预设是保存的设置组合。为每个重复任务构建一个。

效果良好的示例：

- **Web 英雄图像** — WebP，最大宽度 1920，附加 `-web`。
- **客户端可交付成果** — WebP，最大宽度 2560，替换原始内容，保存到“~/Deliverables/”。
- **屏幕录制** — H.264 平衡、带状音频。
- **扫描的 PDF** — 平整、中等质量、灰度。

在**设置 → 预设**中创建它们。每个都可以：

- 适用于所有媒体或仅一种类型（图像/视频/PDF）。
- 使用自己的保存位置和文件名规则。
- 观看其自己的文件夹*（见下文）*。
- 删除元数据，清理文件名，完成后打开输出文件夹。

随时从侧边栏切换活动预设。

## Watch folders

将文件放入文件夹中，让 Dinky 在后台处理它们。

- **全球观看** — *设置→观看→全球*。使用侧边栏当前设置的任何内容。适合“传入”或屏幕截图文件夹。
- **按预设观看** — 每个预设还可以使用自己的规则观看自己的文件夹。独立于侧边栏——随心所欲地更改侧边栏，预设仍然可以发挥作用。

> **专业提示：** 结合“屏幕录制文件夹”+ 一个可剥离音频并重新编码为 H.264 平衡的预设。点击“⌘⇧5”，屏幕录制，点击停止——在您到达 Finder 之前，Dinky 已准备好一个小文件。

## Manual mode

当您想要完全控制时，打开**设置 → 常规 → 手动模式**。

放入的文件不会自动压缩。右键单击任意行即可当场选择格式，当队列准备就绪时使用 **文件 → 立即压缩** (`{{SK_COMPRESS_NOW}}`)，或者首先更改侧边栏中的设置。当一批包含非常不同的文件时很有用。

## Shortcuts app

Dinky 为“快捷方式”应用程序注册了“压缩图像”操作。使用它通过 Dinky 以选定的格式传输 Finder 文件或其他操作 - 与应用内压缩相同的引擎（尊重智能质量、调整大小和元数据的设置）。

## Privacy & safety

- 一切都在**本地运行**。没有上传，没有遥测，没有帐户。
- **崩溃报告**仅在*您*选择发送时发送 - 通过崩溃后提示、“报告错误...”菜单或错误详细信息表。如果您在系统设置中选择了 macOS 诊断共享，Apple 还会通过 MetricKit 代表您向 Dinky 提供匿名崩溃数据，而不会留下任何其他数据离开您的 Mac。
- 编码器（`cwebp`、`avifenc`、`oxipng` 以及 Apple 的内置 PDF 和 AVFoundation 视频管道）内置于应用程序中并直接读取您的文件。
- 默认保留原件。 *压缩后将原件移至垃圾箱*是可选的，位于**设置 → 常规**。
- *如果节省量低于则跳过*（默认情况下关闭）可防止已经精简的文件被无用地重新编码。
- 任何预设中的*剥离元数据*都会删除 EXIF、GPS、相机信息和颜色配置文件。在将照片发布到网络之前值得这样做。

## Troubleshooting

**生成的文件比原始文件大。**
Dinky 则保留原件。您会在该行中看到*“无法将其缩小。保留原始版本。”*。

**文件被跳过。**
要么它已经非常小（低于您的*如果节省低于*则跳过阈值），或者编码器无法读取它。单击该行了解详细信息。

**视频需要很长时间。**
视频重新编码需要大量 CPU 资源。 **设置→常规**中的*批处理速度*设置控制一次运行的文件数量 - 如果您的 Mac 正在执行其他操作，请将其设置为 **快速**。

**我的 PDF 丢失了文本选择/超链接。**
您使用了*压平（最小）*。将预设的 PDF 输出切换为“保留文本和链接”并重新运行。扁平化总是以尺寸取胜；保留总是胜于实用性。

**右键单击“使用 Dinky 压缩”未显示。**
安装后打开 Dinky 一次，以便 macOS 注册该服务。如果它仍然没有出现，请在**系统设置→键盘→键盘快捷键→服务→文件和文件夹**中启用它。

**为什么Dinky不输出JPEG？**
WebP 和 AVIF 绝对优于 JPEG — 相同的视觉质量、更小的文件，并且在所有重要的地方都受到支持。如果您的平台需要 `.jpg`，请先尝试 WebP；现在它几乎已被普遍接受。如果您遇到真正拒绝它的地方，请联系我们并告知我们。
