# Welcome

Dinky는 파일을 엉뚱하게 만듭니다. 무언가를 집어넣고 더 작은 버전을 꺼내세요. 모양은 동일하고 무게는 더 가볍습니다.

**이미지**(JPEG, PNG, WebP, AVIF, TIFF, BMP), **비디오**(MP4, MOV, M4V) 및 **PDF**에서 작동합니다.

모든 일은 Mac에서 이루어집니다. 아무것도 업로드되지 않았습니다.

---

## Quick start

1. 파일(또는 그 더미)을 Dinky의 창으로 드래그합니다.
2. 카운트가 줄어드는 것을 지켜보세요.
3. 다운로드 폴더 또는 선택한 폴더에서 원본(기본값) 옆에 있는 작은 복사본을 찾으세요.

그게 다야. 기본값이 좋습니다. 당신의 의지에 따라 구부리고 싶다면 계속 읽어보세요.

## Ways to compress

먼저 앱을 열 필요는 없습니다. 귀하의 작업 방식에 맞는 것을 선택하십시오.

- **Dinky 창이나 Dock 아이콘으로 드래그 앤 드롭**하세요.
- **파일 열기...** — `{{SK_OPEN_FILES}}`를 사용하여 시트에서 선택합니다.
- **클립보드 압축** — `{{SK_PASTE}}`는 Finder(이미지, 비디오, PDF)에 복사된 지원되는 **파일** 또는 **원시 이미지** 데이터(스크린샷 또는 브라우저의 PNG/TIFF)를 붙여넣습니다.
- **Finder → 서비스 → Dinky로 압축**에서 마우스 오른쪽 버튼 클릭 - 모든 크기 선택에서 작동합니다.
- **폴더 보기** — Dinky는 폴더에 있는 새로운 항목을 모두 압축합니다. (아래 *감시 폴더*를 참조하세요.)
- **빠른 작업** — 시스템 설정 → 키보드 → 키보드 단축키 → 서비스에서 "Dinky로 압축"에 키보드 단축키를 할당합니다.

## Where files go

**설정 → 출력**에서 한 번 설정해 보세요.

- **원본과 동일한 폴더** *(기본값)* — 항목을 깔끔하고 로컬로 유지합니다.
- **다운로드 폴더** — 이메일이나 메시지를 많이 처리하는 경우에 좋습니다.
- **사용자 정의 폴더...** — Dinky를 어디든 가리킵니다.

### Filenames

- **"-dinky" 추가** *(기본값)* — `photo.jpg`는 `photo-dinky.jpg`가 됩니다. 원본은 안전합니다.
- **원본 바꾸기** — 파일을 덮어씁니다. 마지막에 하나의 깨끗한 파일을 원할 경우 **일반**에서 *원본을 휴지통으로 이동*과 결합하세요.
- **사용자 정의 접미사** — 작업 흐름을 조정하는 사람들을 위한 것입니다. 귀하의 파일링 시스템에 적합한 것을 사용하십시오.

> **프로 팁:** 사전 설정은 규칙에 따라 저장 위치와 파일 이름을 재정의할 수 있습니다. "스크린샷 → `~/Desktop/web/`, 원본 교체"와 같은 작업에 사용하세요.

## Sidebar & formats

기본 창 오른쪽에 있는 사이드바는 Dinky에게 **무엇을** 만들 것인지 알려주는 곳입니다.

### Simple sidebar (default)

세 가지 일반 언어 선택: **이미지**, **비디오**, **PDF**. 카테고리별로 하나씩 선택하여 삭제하세요. Dinky는 합리적인 인코더, 품질 및 크기를 파악합니다.

### Full sidebar

**설정 → 일반 → 간단한 사이드바 사용**을 끄거나 개별 섹션을 켜서 모든 컨트롤을 표시합니다.

- **이미지** — 형식, 콘텐츠 힌트(사진/그림/스크린샷), 최대 너비, 최대 파일 크기.
- **동영상** — 코덱 제품군(H.264 / HEVC / AV1), 품질 등급, 스트립 오디오.
- **PDF** — 텍스트 및 링크를 보존하거나 가능한 가장 작은 파일을 위해 이미지로 병합합니다.

## Smart quality

**스마트 품질**이 켜져 있으면(새 사전 설정의 기본값) Dinky는 각 파일을 검사하고 그에 대한 설정을 선택합니다.

- 이미지는 콘텐츠(바쁜 사진과 그래픽 — UI, 일러스트레이션, 로고, 스크린샷)에 맞춰 인코더를 조정합니다.
- 비디오는 해상도와 소스 비트 전송률을 기준으로 계층을 얻은 다음 콘텐츠 유형에 따라 조정됩니다. 화면 녹화 및 애니메이션/모션 그래픽이 계층 위로 이동하여 텍스트와 가장자리를 계속 읽을 수 있습니다. 카메라 영상은 EXIF ​​제조업체/모델로 식별되므로 과도하게 보호되지 않습니다. HDR 소스(Dolby Vision, HDR10, HLG)는 색상을 보존하고 디테일을 강조하기 위해 HEVC로 내보내집니다. H.264는 자동으로 SDR로 평면화합니다.
- PDF는 문서의 복잡성과 텍스트 우선인지 이미지 중심인지에 따라 계층을 얻습니다.

고정된 품질 계층(비디오의 경우 균형/높음, PDF의 경우 낮음/중간/높음)을 원할 경우 **압축**의 사전 설정에서 이 기능을 끄세요. 이는 예측 가능한 결과가 필요한 배치에 유용합니다.

## Presets

사전 설정은 저장된 설정 조합입니다. 반복되는 작업마다 하나씩 구축하세요.

잘 작동하는 예:

- **웹 히어로 이미지** — WebP, 최대 너비 1920, `-web`을 추가합니다.
- **클라이언트 결과물** — WebP, 최대 너비 2560, 원본 교체, `~/Deliverables/`에 저장.
- **화면 녹화** — H.264 밸런스, 스트립 오디오.
- **스캔한 PDF** — 병합, 중간 품질, 회색조.

**설정 → 사전 설정**에서 생성하세요. 각각은 다음을 수행할 수 있습니다.

- 모든 미디어 또는 한 가지 유형(이미지/비디오/PDF)에만 적용됩니다.
- 자체 저장 위치 및 파일 이름 규칙을 사용합니다.
- 자체 폴더를 감시합니다*(아래 참조)*.
- 메타데이터를 제거하고, 파일 이름을 삭제하고, 완료되면 출력 폴더를 엽니다.

언제든지 사이드바에서 활성 사전 설정을 전환할 수 있습니다.

## Watch folders

파일을 폴더에 넣고 Dinky가 백그라운드에서 파일을 처리하도록 하세요.

- **글로벌 감시** — *설정 → 감시 → 글로벌*. 사이드바에 현재 설정된 값을 사용합니다. "수신" 또는 스크린샷 폴더에 적합합니다.
- **사전 설정별 감시** — 각 사전 설정은 자체 규칙에 따라 자체 폴더를 감시할 수도 있습니다. 사이드바와는 별개로 사이드바를 원하는 대로 변경해도 사전 설정이 제 역할을 합니다.

> **프로 팁:** "화면 녹화 폴더" + 오디오를 제거하고 H.264 밸런스로 다시 인코딩하는 사전 설정을 결합합니다. '⌘⇧5' 누르기, 화면 녹화, 중지 누르기 — Dinky는 Finder에 도달하기 전에 작은 파일을 준비했습니다.

## Manual mode

모든 권한을 원할 경우 **설정 → 일반 → 수동 모드**를 켜세요.

끌어온 파일은 자동 압축되지 않습니다. 행을 마우스 오른쪽 버튼으로 클릭하여 즉시 형식을 선택하거나, 대기열이 준비되면 **파일 → 지금 압축**(`{{SK_COMPRESS_NOW}}`)을 사용하거나 사이드바에서 설정을 먼저 변경하세요. 하나의 배치에 매우 다른 파일이 포함되어 있는 경우 유용합니다.

## Keyboard shortcuts

**설정 → 바로가기**에서 동일한 목록을 찾을 수 있으므로 이 페이지를 자세히 살펴볼 필요가 없습니다.

| 바로가기 | 액션 |
| --- | --- |
| `{{SK_OPEN_FILES}}` | 파일 열기... |
| `{{SK_PASTE}}` | 클립보드 압축 |
| `{{SK_COMPRESS_NOW}}` | 지금 압축(큐 실행 - 특히 수동 모드에서 유용함) |
| `{{SK_CLEAR_ALL}}` | 모두 지우기 |
| `{{SK_TOGGLE_SIDEBAR}}` | 형식 사이드바 전환 |
| `{{SK_DELETE}}` | 선택한 행 삭제 |
| `{{SK_SETTINGS}}` | 설정 |
| `{{SK_HELP}}` | 이 도움말 창 |

**시스템 설정 → 키보드 → 키보드 단축키 → 서비스**에서 *Dinky로 압축*에 대한 고유한 내용을 추가하세요.

## Shortcuts app

Dinky는 바로가기 앱에 **이미지 압축** 작업을 등록합니다. 이를 사용하여 선택한 형식(인앱 압축과 동일한 엔진)을 사용하여 Dinky를 통해 Finder 파일 또는 기타 작업을 파이프할 수 있습니다(스마트 품질, 크기 조정 및 메타데이터에 대한 설정 참조).

## Privacy & safety

- 모든 것이 **로컬**에서 실행됩니다. 업로드도, 원격 측정도, 계정도 없습니다.
- **충돌 보고서**는 *사용자*가 선택한 경우에만 전송됩니다. — 충돌 후 프롬프트, "버그 보고..." 메뉴 또는 오류 세부 정보 시트를 통해. 시스템 설정에서 macOS 진단 공유를 선택한 경우 Apple은 Mac에서 추가 데이터 없이 MetricKit을 통해 사용자를 대신하여 익명의 충돌 데이터를 Dinky에 전달합니다.
- 인코더(`cwebp`, `avifenc`, `oxipng`와 Apple의 내장 PDF 및 AVFoundation 비디오 파이프라인)가 앱 내부에 탑재되어 파일을 직접 읽습니다.
- 원본은 기본으로 보관됩니다. *압축 후 원본을 휴지통으로 이동*은 **설정 → 일반**에서 선택할 수 있습니다.
- *아래 저장 시 건너뛰기*(기본적으로 꺼져 있음)는 이미 간결한 파일이 아무 이유 없이 다시 인코딩되지 않도록 보호합니다.
- 모든 사전 설정의 *스트립 메타데이터*는 EXIF, GPS, 카메라 정보 및 색상 프로필을 제거합니다. 사진을 웹에 게시하기 전에 시도해 볼 가치가 있습니다.

## Troubleshooting

**원본보다 파일이 크게 나왔습니다.**
Dinky는 대신 원본을 유지합니다. 행에 *"이 항목을 더 작게 만들 수 없습니다. 원본을 유지하세요."*가 표시됩니다.

**파일을 건너뛰었습니다.**
이미 매우 작았거나(절감액이 임계값보다 *낮으면 건너뛰기*) 인코더가 이를 읽을 수 없었습니다. 자세한 내용을 보려면 행을 클릭하세요.

**동영상을 보는 데 시간이 오래 걸립니다.**
비디오 재인코딩은 CPU를 많이 사용합니다. **설정 → 일반**의 *배치 속도* 설정은 한 번에 실행되는 파일 수를 제어합니다. Mac에서 다른 작업을 수행하는 경우 **빠름**으로 설정하세요.

**내 PDF에서 텍스트 선택/하이퍼링크가 손실되었습니다.**
*평탄화(가장 작음)*를 사용했습니다. 사전 설정의 PDF 출력을 *텍스트 및 링크 유지*로 전환하고 다시 실행하세요. Flatten은 항상 크기 면에서 승리합니다. 보존은 항상 유용성에서 승리합니다.

**"Dinky로 압축"을 마우스 오른쪽 버튼으로 클릭하면 표시되지 않습니다.**
macOS가 서비스를 등록하도록 설치 후 Dinky를 한 번 엽니다. 그래도 표시되지 않으면 **시스템 설정 → 키보드 → 키보드 단축키 → 서비스 → 파일 및 폴더**에서 활성화하세요.

**Dinky가 JPEG를 출력하지 않는 이유는 무엇입니까?**
WebP와 AVIF는 JPEG보다 훨씬 우수합니다. 동일한 시각적 품질, 더 작은 파일, 중요한 모든 곳에서 지원됩니다. 플랫폼에 `.jpg`가 필요한 경우 먼저 WebP를 사용해 보세요. 이제는 거의 보편적으로 받아들여지고 있습니다. 진심으로 거부하는 곳을 방문했다면 연락해서 알려주세요.
